入户作业全流程（6 阶段）

从晨会到离开后回单，每个阶段的关键动作和输出物。

01

晨会

  • 点名、身体状况、是否饮酒
  • 工装/工号牌/登高证检查
  • 绝缘鞋、绝缘手套、电笔、安全帽验电
  • 小工具、熔接机、光功率计、急救包、安全带、梯子
  • 四必讲：网格工单 → 安全事故案例 → 安全知识抽问 → 维护指标
  • 电瓶车检查后出发
02

预约调度

  • 工单分派、用户情况、三推目标识别
  • 主动改约 / 被动改约协同
  • 耳机/CP 自动提醒首次/二次/三次
  • 确认客户在家、户型、工具与业务资料
03

入户前

  • 核对客户姓名、套餐档次、5GA / W话返资格
  • 带齐 FTTR、千兆路由、机顶盒、看家摄像头物料
  • 提前看一眼"四看"信息：设备 / 环境 / 人群 / 行为
  • 开启工单任务，准备开口话术
04

作业中

  • 开口话术：检测切入 / 普遍现象 / 福利切入
  • 核实户型与网口布设，判断明线/暗管
  • 测速、光功率（-8 ~ -25 dBm）、DNS、终端、路由器
  • 故障排查走"故障诊断树"分支
  • 展示热力图与房间测速制造体验差距
05

离开前

  • 讲解小贴纸、IPTV/会员/回看演示
  • FTTR / 千兆路由 / 移动看家随销引导
  • 异议处理三类：太贵 / 够用 / 考虑考虑
  • 满意度提醒，添加企业微信
  • 工具设备清点回收
06

离开后

  • 当场或赶回前完成回单
  • 拍照归档：进户、设备、测速、签字
  • 商机字段沉淀：5GA、FTTR 候选、智能家居预留
  • 投诉/质差类工单复盘到群

晨会标准脚本

来源：晨会流程zjl.docx，按"工装 → 工具 → 四必讲 → 安全抽问 → 天气与维护指标"逐项打卡。

① 工装与人员检查

  1. 点名（应到/实到/请假），询问昨夜饮酒、身体不适
  2. 工装统一、腰包、急救包、防暑用品到位
  3. 工号牌 + 登高证（在有效期内）
  4. 绝缘鞋（无破损/裂痕）
  5. 绝缘手套（无破损，戴好）
  6. 测电笔（外观完好、有电）
  7. 安全帽（帽檐/帽寸/下颌带、近电告警器、延长线）
  8. 戴好手套与安全帽 → 远→近 30cm 验电（先帽后笔）

② 生产工具检查

  1. 小工具齐全完好、绝缘处理
  2. 熔接机 / 光功率计 / 红光源电量充足
  3. 酒精与酒精棉签是否需补充
  4. 急救包药品有效期
  5. 防暑用品（藿香正气水）按季节抽检
  6. 安全带 / 安全绳 拉力测试
  7. 梯子防滑垫、防滑条、反光条、抖动检测

③ 早交班"四必讲"

  1. 工单与风险源：今日网格 X 单装机/Y 单四级一般工单；弱电井/共建机房黑暗作业、不明线物按电力线处理、二次作业必须再验电
  2. 安全事故案例：选一例（岳阳触电、洛阳杆倾倒），强调近电规范、登梯登杆四必做、登高五必做
  3. 安全知识抽问：登梯/登杆四必做、登高五必做、一级风险源定义、六严禁、高/低压线根数
  4. 维护指标提醒：满意度、样本量、专项、工单积压、随销

④ 登梯登杆"四必做"

登梯：架梯前先验电 → 登梯前再验梯 → 不明线物反复验 → 重复登梯要验电

登杆：登杆前先验电 → 观察杆体无裂变 → 推动杆体做测试 → 离地 30cm 测脚扣

⑤ 登高"五必做"

  1. 观察现场环境，选择合适路径
  2. 安全带打牢，防止高空坠落
  3. 不明线路和物均反复验电
  4. 靠近电线作业保持安全距离
  5. 杆/梯上作业点安全绳挂牢靠

⑥ 作业"六严禁"

  1. 严禁在不具备安全作业环境下作业或施工
  2. 严禁无登高证人员登高作业
  3. 严禁登高作业中单人作业
  4. 严禁抢工期、赶进度违章作业
  5. 严禁违法违章违规指挥和派单
  6. 严禁无工单作业或施工

⑦ 一级风险源识别

  1. 作业空间不能满足要求、危及人身安全
  2. 分线盒/光交箱距变压器或高压线 < 2.5 米
  3. 分线盒在电力线上方且距离较近
  4. 用户线必须经过变压器才能完成作业
  5. 用户线与电力线交越或上方平行距离过近

⑧ 一看二想三拍四操作

看：现场环境、进户点、径路风险源

想：劳保用品、避开/防范风险源（一级停作业上报）

拍：佩戴齐全后拍照上传安全群

操作：登梯登杆双十条、规范近电作业、遇一级风险源立即停作业

故障诊断树（5 大场景）

按用户现象逐层细化 → 排查方向 → 现场处置步骤。来源：4.26 家庭宽带网络中断排查要点.xlsx。

① 网页打不开

  • 所有网页打不开，QQ/微信正常：DNS 问题 → 检查光猫/路由 DNS → 改为 112.4.0.55 → 重启光猫与路由 → 重测
  • 所有网页+QQ 都断：光猫 LOS 红灯 → 查光功率/断纤 → 重插光纤、重启 → 修复外线
  • 仅部分打不开：代理/VPN 问题 → 关代理 → 清缓存 → 重置网络 → 路由出厂
  • 单一网页打不开：用 4G/5G 测同一网页 → 若也不通则站点问题；若通则查防火墙、MTU；用 itdog 三网测试，定位集团/反诈/公安封堵

② 游戏无法连接

  • 所有游戏登不上：DNS/代理 → 检查 DNS 是否 112.4.0.55 → 关代理 → 重启光猫/路由
  • 能登录但进不了对局：端口/协议被阻 → 关 SPI 防火墙 → 开 UPnP → 设置 DMZ → 用游戏加速器
  • 单一游戏不通：查游戏官方服务器；改路由 MTU 1400-1450
  • 对局中频繁掉线：查 lost-carrier、光功率、PON 口稳定、路由过热 → 换网线/路由
  • 时延大：用 IP 雷达抓游戏服 IP → tcping 看互通 → 异网走优化线路
  • 登录成功无法组房：NAT4 → 改为 NAT1

③ 直播无法观看 / 推流

  • 所有平台加载失败：DNS/代理；关 IPv6；关广告拦截；换 APP/浏览器
  • 有声无画：带宽/解码 → 测速、降画质、换 APP、关省流、查路由 QoS
  • 推流中断：查 lost-carrier、光功率、PON、法兰头/分光器、必要时换光猫尾纤
  • "主播网络不稳定"：测上行；PON 口建议 ≤32 户；关上行占用设备（监控/云盘）
  • "连接服务器失败"：切线路、换协议（RTMP→SRT）、手机热点对照
  • 推流端口被阻：开放 1935/443，开 UPnP 或端口转发，关 SPI

④ 视频无法播放

  • 所有平台不能播：查光猫灯、测速、DNS、关代理、重启路由
  • 能加载列表点不开：解码/APP → 关省流模式、清缓存、更新 APP
  • 单一平台不通：异网测同站；改移动 DNS；VPN 测限流
  • 部分视频不通：抓包分析 IP/域名 → 公安/GFW 封堵或质差

⑤ 频繁断网，时断时续

  • IP 冲突：查路由内是否配静态 IP → 改 DHCP 自动 → 重启路由+冲突设备 → 清 ARP 缓存
  • NAT 会话超限（智能终端异常）：拔掉异常终端或换设备
  • NAT 会话超限（多人办公）：改 NAT1，建议改极速一网通专线（动态公网 IP，无会话数限制）
  • 流氓 ONU：同一 PON 多用户掉线 → 后台查、现场反向收光定位
  • 光功率临界：掌上家客查询临界值，处理外线/法兰头

兜底排查清单

  • 后台账号状态 + 光功率（参考区间 -8 ~ -25 dBm）
  • 远程或手动重启光猫
  • 路由器年龄 > 2 年 / Wi-Fi5 以下 → 评估更换
  • 仍异常考虑更换设备或室内布线整改

场景话术库

开口 → 转折 → 价值 → 异议 → 促成，按场景挑卡组合。

入户开口（四层结构）

① 开场白（建立信任）

  • 检测切入："我今天主要帮您看看路由器性能，顺便做个网络检测。"
  • 普遍现象："很多用户一开始觉得网够用，一测才发现路由器跟不上了。"
  • 专业性："您家这个户型我见过挺多次，信号盲区一般在卧室或阳台。"
  • 福利切入："我顺便帮您查一下系统，看有没有还没领的免费升级或换新优惠。"

② 转折句（重新定义问题）

  • "其实这不是宽带的问题，是路由器有点跟不上了。"
  • "您可能没注意到，但这个会影响您看视频、打游戏的稳定性。"
  • "现在用着行，但以后家里设备多了，可能就容易卡。"
  • "我有个客户跟您家情况差不多，换了路由器后稳多了。"

③ 引导未来 / 价值呈现

  • 客厅 500M、小房间不到百兆 → 家人多、孩子上网课会卡
  • FTTR 全屋满格、无缝漫游、低延迟、智能家居预留
  • "换成 FTTR 走到哪都流畅，您也不用天天操心网络。"

④ 促成（降低门槛 + 二选一）

  • "今天正好有个活动，我帮您申请一下，挺划算的。"
  • "我先给您装上，您试用几天感受一下，不合适我一个电话过来拆，一分钱不收。"
  • "现在升级，相当于提前解决了以后可能出现的卡顿问题。"
  • "您看是今天装方便，还是明天装方便？"

FTTR 推荐 6 场景

5GA 目标用户（免费送）

适用：主套餐高，符合 5GA 体验条件，免 200 调试费。

"您号码套餐挺高，符合 5GA 体验条件。资费不变（或微涨），可以领一套 FTTR 一主一从，比现在路由器强多了。"

W 话返目标用户（免费送）

适用：199 元档返 30 元 / 150 元档返 25 元。

"充值 200 元送 FTTR 一主一从。非千兆用户：办理 FTTR 充 200 同步免费升千兆。"

正常收费月包（百兆/大户型）

适用：百兆路由、面积大、网速慢用户，以"二次光改"切入。

"千兆宽带配百兆路由跟不上，每个月多花一包烟钱，一步到位每个房间满格。"（配老客户案例图）

焕新套餐升档

适用：可升档至全家享 89 元及以上。

"换 89 元全家享，每月 20 元权益金可抵 FTTR 设备费，不花啥钱把家网彻底升级。"

存量接续（千兆 + 5 元）

适用：现千兆 20 元、消费 60 元以上。

"每月加 5 元（共 25 元）+ 充 200，一步到位升级 FTTR 一拖一高端路由。"

比算新增 / 异网迁入

适用：家人有异网卡或新增需求。

"办 109/169/199 比算套餐，比分开用便宜，自带 FTTR 设备。"

异议处理（三类拒绝）

价格敏感型："太贵了"

共情 → 重构价值 → 拆分成本

"完全理解。但您路由器用了 3 年多，百兆跑不满 300M，小马拉大车。FTTR 设备不收费，每月只付千兆提速费用，每天不到 1 元，全屋满格。"

"两台好路由也要四五百，用一两年又卡。FTTR 一次 200、月付 25，包安装包调试，协议期内坏了免费换。"

满足现状型："够用"

挖隐性痛点 → 描绘场景 → 制造体验差距

"周末窝沙发刷视频突然转圈过吗？家里 2 部手机+电视+平板+智能家居同时在线，百兆路由就像一条小路。"

"FTTR 给每个房间单独一条光纤高速路，互不干扰。"（现场打开 Wi-Fi 测速 + 热力图佐证）

拖延决策型："考虑考虑"

给理由 + 制造稀缺 + 降低门槛

"换网络确实要好好考虑。这个'老用户免费升级 FTTR'活动到月底就结束，您可以先占名额。"

"今天先给您装上试用 7 天，不满意一个电话拆走，一分钱不收。今天办还能申请优先安装。"

高潜用户"四看"识别

看设备

  • 路由器 > 2 年
  • Wi-Fi5 以下 / 百兆款
  • 老款机顶盒
  • 套餐 < 300M

看环境

  • ≤80㎡ 单路由
  • 80-120㎡ 双路由 / FTTR
  • >120㎡ 强推 FTTR
  • 关注卧室/书房/卫生间/阳台

看人群

  • 游戏玩家：FTTR + 电竞加速
  • 网课家庭：FTTR 全屋覆盖
  • 商铺/办公：FTTR + 安防
  • 老年用户：大屏会员 + 看家

看行为

  • 多设备同时在线
  • 高峰期用网
  • 有投诉/报修历史
  • 多意图/质差工单

投诉预处理工具链

来源：苏州移动（内容）投诉预处理.docx — 命令模板可现场复制使用。

ping

cmd 内置
ping www.baidu.com
ping -t 1.1.1.1
ping -4 -n 20 host
ping -l 1400 -f host

常用项：-t 持续 / -a 反查主机名 / -4/-6 强制版本 / -n 次数 / -l 缓冲 / -f 不分段（仅 IPv4） / -i TTL

tracert

cmd 内置
tracert www.baidu.com
tracert -d -h 30 host
tracert -4 -w 2000 host

常用项：-d 不解析主机名 / -h 最大跃点 / -w 等待 ms / -4/-6

nslookup

cmd 内置
nslookup baidu.com
nslookup baidu.com 112.4.0.55

查域名解析；可指定 DNS。

tcping

外部工具
tcping 183.232.231.172 80
tcping -n 20 host 443
tcping -i 0.5 -w 2 host 80

测 TCP 端口（ping 走 ICMP 测不了端口）。port is open / socket is not connected。

dig

需安装
dig baidu.com
dig baidu.com @112.4.0.55
dig baidu.com +short
dig MX baidu.com
dig -x 39.134.69.205
dig baidu.com +trace
dig baidu.com aaaa

比 nslookup 强，支持 DNS 指定、PTR 反查、解析溯源、IPv6。

besttrace

路由追踪

GUI 工具，输入目标 IP 即可看到路由路径，可视化定位质差/绕行。

Mac/iOS 在 App Store 下载；Windows/Linux 见资料原文链接。

IP 雷达 5

使用说明

现场人员按这个流程采集 SOP 穿越数据，开发人员按这个流程发布新版本。

1. 登录人员

进入页面后先选择默认人员或手工填写姓名、手机号、单位和角色。后续新增的场景、现场记录、录音和问题都会自动带上当前人员。

2. 场景维护

在“场景清单”里可以新增、查询、修改和删除场景。重点维护目标场景、场景描述、录音文件名、关键词、需采集数据和开发数据支撑点。

3. 现场记录

每跑一轮穿越，在“现场记录”新增一条。录音文件名要和录音关键词里的文件名保持一致，便于后续用大模型分析。

4. 录音上传、播放和下载

在“录音关键词”点击新增录音，选择本地 .mp3 文件并保存。保存后可以在表格中直接播放，也可以点击下载。音频文件保存在当前浏览器本地数据库，不会进入 CSV。

5. 问题与字典

现场发现的知识、流程、话术、接口问题记录到“问题需求”。下拉选项在“字典表”维护，每行一个选项，保存后全站同步。

6. 数据导出

右上角支持导出 JSON 和 CSV。JSON 用于完整备份浏览器里的结构化数据，CSV 用于给团队汇总分析。音频文件需通过页面中的下载入口单独保存。

7. GitHub 地址

本系统代码统一维护在 GitHub 仓库。后续功能修改、版本记录和服务器部署都以该仓库为准。

https://github.com/laozhuclaw/sop
